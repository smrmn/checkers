# Шашки с модификаторами — CONTEXT.md

## Продукт
Локальное веб-приложение, один HTML-файл. Чистый HTML/CSS/JS, без фреймворков и зависимостей.

---

## Файловая структура

```
checkers.html          ← собранный файл (открывать в браузере, коммитить)
build.py               ← сборщик: python3 build.py
src/
  template.html        ← HTML-скелет с маркерами /* INJECT:path */
  css/
    style.css
  js/
    01_state.js        ← gameState
    02_board.js        ← initBoard, getMovesForPiece, getCapturesForPiece, checkGameEnd
```

## Визуал и UI

- Доска 8×8, **чёрно-белая** классическая
- Шашки игрока — **синие**, бота — **красные**
- Без дамок
- Под доской — **футер с двумя тогглами** (по умолчанию OFF):
  - "Хрупкие плитки"
  - "Туман войны"
- Тогглы активны **только до старта и после окончания** матча (`opacity: 0.28` + `pointer-events: none` в состоянии `.playing`)
- Таймер запускается с первым ходом игрока

## Механика ходов (click-based, без drag & drop)

1. Клик на свою шашку → она **выделяется** (белый outline), доступные клетки **не подсвечиваются**
2. Клик на доступную клетку → **ход выполнен**, выделение снято
3. Клик на **недоступную клетку** → ничего не происходит, выделение **остаётся**
4. Клик на другую свою шашку → переключает выделение на неё
5. **Анимации — нет. Drag & drop — нет.**
6. Шашки, которые обязаны бить, обводятся **красным кольцом** (`.must-capture`)

---

## Правила игры

- Классические шашки: ход по диагонали вперёд на 1 клетку
- **Мульти-поедание**: если после взятия есть ещё возможность бить — ход продолжается той же шашкой (обязательно)
- Взятие — во все 4 диагональных направления (не только вперёд)
- **Дамок нет**
- Шашка, достигшая последней линии противника, **не превращается**
- **Конец игры**:
  - Все шашки противника съедены → победа
  - Нет доступных ходов → конец, побеждает сторона с большим кол-вом шашек
  - Одинаковое кол-во → **ничья**

## Дополнительные механики (тогглы)

### 1. Хрупкие плитки (Fragile Tiles)
- Каждая **тёмная** клетка хранит счётчик уходов: `0` → `1` → `2`
- **Стартовые клетки** (на которых шашка стоит изначально) — не трескаются при первом уходе.
  Клетка помечается `hasReceivedPiece = true` только когда на неё **приходит** шашка ходом.
- После **первого** ухода (после прихода): клетка **треснута** (amber диагональные линии)
- После **второго** ухода: клетка **уничтожена** — недоступна для хода и для приземления
- Уничтоженные клетки: тёмный фон + символ `×`
- Механика применяется к обоим сторонам

### 2. Туман войны (Fog of War)
- Активен **после первого хода игрока** (до этого вся доска видна)
- Игрок видит только клетки в **радиусе 2** (чебышёвское расстояние) от любой своей шашки
- Всё остальное скрыто: **единая SVG-маска** (`#fog-layer`) с серым цветом `rgba(90,90,100,0.92)`
  и скруглёнными краями видимых зон (`rx="48"`)
- SVG имеет `pointer-events: none` — не блокирует клики по клеткам
- **Бот видит всё** — играет по полной информации
- Шашки бота в тумане **не отображаются**

## Логика бота (простой ИИ)

Приоритеты при выборе хода:

1. **Съедение** — если есть хотя бы одно взятие, бот **обязан** бить (как по правилам)
2. **Мульти-съедение** — выбирает **самую длинную цепочку** (`calcMaxChain` — рекурсивная симуляция)
3. **Избегать опасных клеток** — не ходить на клетки, откуда его шашку сразу могут съесть
4. **Избегать треснутых клеток** — при прочих равных не становиться на `cracked`-клетки
5. **Случайный ход** — если всё равнозначно

Задержка хода бота: **600мс**.

## Экран завершения матча

При окончании матча доска затемняется на **80%** (`rgba(0,0,0,0.80)`) через `#end-overlay`.
По центру доски — вертикальный блок:
1. Время матча (`#end-time`)
2. Результат (`#end-result`): "Победа игрока" / "Победа бота" / "Ничья"
3. Оставшиеся шашки: "Синих: N · Красных: N" (`#end-pieces`)
4. Активные механики: чипы `.mod-chip` (`#end-modifiers`) — скрыты если нет активных
5. Кнопка **Reset** (`.btn-reset-overlay`) — по центру, вызывает `resetMatch()` → idle

Кнопка **Reset** в футере — возвращает в idle во время матча.
Кнопка **Start Match** (в idle) — начинает новый матч.

## Структура состояния

```javascript
gameState {
  board[8][8]: {
    piece: null | { color: 'player'|'bot' }
    tileState: 'normal' | 'cracked' | 'destroyed'
    hasReceivedPiece: boolean   // true после первого прихода шашки (для Fragile Tiles)
  }
  currentTurn: 'player' | 'bot'
  selectedPiece: {row, col} | null
  availableMoves: [{row, col}] | [{row, col, captured: {row,col}}]
  inMultiCapture: boolean
  multiCapturePiece: {row, col} | null
  gameStatus: 'idle' | 'playing' | 'finished'
  winner: 'player' | 'bot' | 'draw' | null
  timer: seconds
  timerInterval: id | null
  stats {
    playerCaptured: number
    botCaptured: number
    startTime: timestamp
  }
  toggles {
    fragileTiles: boolean
    fogOfWar: boolean
  }
}
```

## Важные детали реализации

- `startMatch()` читает состояние тогглов **из DOM-чекбоксов** в момент старта — не из `gameState.toggles`.
  Это защита от рассинхрона (браузерное восстановление формы, прямая запись в state и т.п.).
  `gameState.toggles` на протяжении матча = снимок DOM на момент `startMatch()`.

## Известные особенности

- Туман рендерится одним `#fog-layer` SVG поверх доски; не перерисовывает DOM клеток
- SVG маска: белый прямоугольник (весь туман) — чёрные прямоугольники с `rx="48"` (дыры = видимые зоны)
- Тогглы фиксируются в момент старта и не меняются в процессе матча
- Бот не может ходить на `destroyed`-клетки (учитывается в `getCapturesForPiece` / `getMovesForPiece`)
